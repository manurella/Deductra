# Event Protocol and Store

Last reviewed: 2026-07-16

CR-002 establishes the append-only evidence substrate for M1. It defines canonical event envelopes, lifecycle payloads, deterministic integrity verification, a technology-neutral repository port, and a transactional SQLite adapter. It does not calculate puzzle state or decide whether a deduction is valid.

## Canonical envelope

Every event records its event, trace, puzzle revision, branch, sequence, schema version, producer, causation, correlation, timestamp, payload, previous hash, and event hash. Sequence numbers are zero-based and contiguous within a trace. A new trace starts with 64 zeroes as its previous hash.

`event_type` must equal the discriminated payload `kind`. Timestamps require an explicit timezone. Timestamps support audit and ordering but do not give an event logical authority.

The v1 payload union contains trace lifecycle events whose contracts are established in this packet: trace start, puzzle validation, initial-state creation, trace completion, and trace failure. Deduction, verification, search, interaction, generation, reporting, and agent payloads are added only by the packets that own those semantics.

## Integrity

An event hash is the SHA-256 digest of the canonical envelope excluding only `event_hash` itself. Verification checks:

1. contiguous sequence numbers;
2. linkage to the previous event hash;
3. canonical event hashes;
4. one trace identifier and puzzle revision throughout the stream.

After the first invalid event, that event and every later event are reported as untrusted. This is tamper evidence, not encryption, authentication, or a digital signature.

## Persistence boundary

`EventStore` is the repository port. `SQLiteEventStore` is its first adapter and uses the Python standard library only. Appends use an explicit immediate transaction, validate the existing stream, enforce optimistic stream-head continuity, and atomically update the event and stream head. The database enables foreign keys, uses strict tables, and enables write-ahead logging for file-backed stores.

SQLite stores the canonical envelope JSON alongside indexed identity, ordering, and hash columns. Reads reject disagreement between those representations. Migrations are recorded in `schema_migrations`; breaking database changes require additive migration planning and a new architecture decision.

## Replay boundary

CR-002 proves stable ordered event replay and persistent round-trip equivalence. State-level deterministic reduction and replay equivalence remain CR-003 responsibilities because no reducer exists in CR-002.

The public schema is `schemas/event-envelope-v1.schema.json`. It is generated by `scripts/export_json_schema.py` and protected by a drift test.
