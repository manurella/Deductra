# State Reduction and Replay

Last reviewed: 2026-07-16

CR-003 derives immutable puzzle projections from verified canonical events. The event history remains authoritative; state, branch projections, and snapshots are deterministic derivatives that can be discarded and rebuilt.

## Puzzle state

`PuzzleState` records one puzzle revision and branch, the latest state-changing event sequence, candidate domains, asserted and rejected atoms, active constraints, contradictions, completion status, and a canonical SHA-256 state hash.

Candidate collections, atoms, constraint identifiers, and contradiction identifiers are immutable. Canonical serializers sort unordered collections before hashing. State construction rejects structural inconsistencies, including overlapping asserted and rejected atoms, unexplained empty candidate domains, missing singleton assignments, incorrect completion status, and hash drift.

`create_initial_state` projects declared variable domains, domain constraints, and givens. It does not infer new deductions or invoke a solver. `validate_state` checks the projection against the immutable `PuzzleSpec`, including domain membership, assignment consistency, active constraint references, and declared all-different constraints.

## Pure reduction

`reduce_state(previous, event)` returns a new state and never edits its input. CR-003 assigns state semantics only to:

- candidate elimination;
- value assignment;
- contradiction detection.

Every accepted event must pass event-integrity verification, reference the same puzzle revision and branch, advance the branch sequence, and name the exact source-state hash. Candidate and assignment payloads preserve their reasoning origin. Proof verification is not performed here and remains a later package boundary.

Applying the same event to the same state produces byte-identical canonical JSON and the same state hash.

## Retained branches

`BranchProjection` stores one latest state and lifecycle record for every branch. Opening a branch copies the named parent projection and records the assumption, source hash, method, and sequence. Closing a branch changes projection focus to its parent without deleting the child state.

A contradicted branch remains replayable after its parent continues. Search branches require every candidate or assignment mutation to declare `search` origin; search-origin mutations are rejected outside an explicit search branch. This prevents search-derived state changes from being represented as human-rule deductions.

## Replay and snapshots

Replay first verifies the complete event hash chain and matches the supplied genesis state to its `InitialStateCreated` event. It then applies later events in order to rebuild the complete retained-branch projection.

`StateSnapshot` records the state, source event identity and hash, and its own canonical digest. Snapshots are integrity-protected replay accelerators only. They never replace event history as authority, and persistent snapshot storage is not introduced in CR-003.

The public state schema is `schemas/puzzle-state-v1.schema.json`. Event payload additions remain represented by `schemas/event-envelope-v1.schema.json`; both are generated by `scripts/export_json_schema.py` and protected by drift tests.

## Boundary

CR-003 does not determine whether a proposed deduction is logically valid. It adds no verification backend, solver, generator, puzzle-family adapter, UI, report subsystem, learning memory, or agent behavior.
